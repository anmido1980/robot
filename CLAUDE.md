# Robot MVP

Алгоритмическая торговля фьючерсом Si на ММВБ через брокера Т-Инвестиции (gRPC API) в sandbox-режиме.

## Контекст проекта

MVP — работающий repeatable paper-trading pipeline для стратегии Donchian Breakout v2 (Turtle 20/10 + ADX>15) на таймфрейме 1 час.

Брокер: **Т-Инвестиции (Т-Банк)**. API: gRPC (`tinkoff-invest-python`).
Режим по умолчанию: **paper-trading в песочнице**.

## Стек и зависимости

- **Язык:** Python 3.11+
- **Основные пакеты:** `tinkoff-investments`, `pandas`, `numpy`, `pydantic`, `loguru`, `python-dotenv`, `pytest`
- **Брокерский API:** только T-Invest gRPC.
- **Песочница:** обязательна до live.
- **Секреты:** только в `.env` (вне репозитория). Шаблон — `.env.example`.
- **Версии пакетов:** фиксируем в `requirements.txt`.
- **Часовой пояс:** все timestamps в UTC+3 (МСК), внутреннее хранение — ISO 8601.

## Ключевые ограничения (бизнес)

- **Размер позиции:** ≤ 50% депозита.
- **Дневной убыток:** ≤ 5% портфеля. Сброс счётчика после 00:30 МСК следующего дня (клиринг).
- **Kill-switch:** обязателен (ручной — файл-флаг; авто — при −5% за день).
- **Идемпотентность заявок:** через `order_request_id` (UUID) на каждом `PostOrder`.
- **Торговля только инструментами ММВБ** (срочный рынок, `class_code=SPBFUT`).
- **Live-торговля:** только после paper-trading в песочнице ≥ 4 недель с расхождением P&L ≤ 30% от бэктеста.
- **Стартовый live-размер:** 0.5 от целевого.

## Архитектурные принципы

- Слои изолированы: `broker → risk → strategy → runner`. Стратегия не знает о T-Invest API.
- Все модели — pydantic. Никаких dict'ов в бизнес-логике.
- Событийная шина (`core/events.py`) — единственный канал передачи данных между слоями.
- Любая внешняя зависимость (gRPC, файлы, время) — через интерфейс в `core/`, чтобы можно было мокать в тестах.
- Все заявки проходят через `risk/manager.py` до отправки на биржу. Никаких обходных путей.
- Логи — в `06-logs/`, журнал сделок — в SQLite.
- Никакой автоматической торговли в часы клиринга (23:50 до 00:30 МСК) и в аварийных режимах биржи.

## Правила безопасности

1. Не передавать персональные и личные данные в открытом виде в интернет.
2. Обезличивать информацию, относящуюся к владельцу проекта, в файлах, которые передаются нейросети.
3. Git push — только после прямого указания.

## Правила работы (общие)

- Агент сначала показывает план — владелец подтверждает — затем выполняет.
- Результат каждого шага сохраняется в файл.
- Действия с файлами внутри `Robot/` — без подтверждения, кроме удаления.
- Перед удалением файла в `04-output/` или `05-bots/` сохранять копию `.bak`.
- Выход за пределы `Robot/` — только по явному запросу.
- Непроверенная информация — так и обозначаем.
- Работаем только с фактами; прогнозы — отдельно подчёркиваем.
- Язык — русский; термины и код — английский.
- Результаты задач сохраняй в папке `reports`.
